# F5 (BIG-IP)

**Scope: F5 BIG-IP (APM / LTM / ASM).** BIG-IP APM functions as an SSL VPN and access gateway; the load balancer and WAF roles are also in scope where the same management plane is exposed. *(6 edge KEV entries, 2020–2026.)*

## Market Position

F5 BIG-IP is a dominant application delivery controller (ADC) in enterprise and government environments, widely deployed for load balancing, SSL offload, WAF, and — most relevant here — remote access via BIG-IP APM (Access Policy Manager). APM's SSL VPN capability places it directly in the edge security perimeter alongside dedicated VPN appliances. The scope question is legitimate: BIG-IP is not primarily marketed as a firewall, but its management plane exposure and APM role make it an edge device for practical purposes. F5 holds significant share in regulated industries including federal agencies and financial services, which elevates the impact of any management-plane compromise.

## Key Incidents

### CVE-2020-5902 — TMUI RCE (KEV: 2021-11-03)

[CVE-2020-5902](https://nvd.nist.gov/vuln/detail/CVE-2020-5902) (CVSS 9.8) is one of the highest-profile F5 vulnerabilities on record. The flaw — a path traversal allowing unauthenticated remote code execution via the Traffic Management User Interface (TMUI) — was disclosed by F5 on July 1, 2020. Mass exploitation began almost immediately: within days, the [NSA, CISA, and multiple threat intel firms](https://www.cisa.gov/news-events/alerts/2020/07/24/mitigating-recent-threat-actors-activity) reported widespread scanning and exploitation by nation-state actors and ransomware groups. The CISA KEV entry (November 2021) trails the actual exploitation peak by over a year — the real-world urgency was immediate in July 2020.

### CVE-2021-22991 — TMM Buffer Overflow (KEV: 2022-01-18)

[CVE-2021-22991](https://nvd.nist.gov/vuln/detail/CVE-2021-22991) is a buffer overflow in BIG-IP's Traffic Management Microkernel (TMM). While technically distinct from the management UI, the vulnerability is reachable without authentication and represents continued risk in a core data-plane component.

### CVE-2022-1388 — iControl REST Auth Bypass (KEV: 2022-05-10)

[CVE-2022-1388](https://nvd.nist.gov/vuln/detail/CVE-2022-1388) (CVSS 9.8) is the most operationally significant F5 vulnerability of 2022. The flaw allows an unauthenticated attacker to bypass authentication to the iControl REST API and execute arbitrary commands with root privileges. F5 disclosed it on May 4, 2022. Within **two days** of public PoC release, [CISA, Horizon3.ai, and multiple threat intel teams](https://www.cisa.gov/news-events/alerts/2022/05/18/threat-actors-exploiting-f5-big-ip-cve-2022-1388) documented mass exploitation across thousands of organizations worldwide. The exploitation speed — PoC to mass compromise in 48 hours — puts CVE-2022-1388 among the fastest-weaponized edge vulnerabilities on record.

### CVE-2023-46747 and CVE-2023-46748 — BIG-IP Configuration Utility Chain (KEV: 2023-10-31)

[CVE-2023-46747](https://nvd.nist.gov/vuln/detail/CVE-2023-46747) is an authentication bypass in the BIG-IP Configuration Utility (CVSS 9.8) that, when chained with [CVE-2023-46748](https://nvd.nist.gov/vuln/detail/CVE-2023-46748) (an authenticated SQL injection), enables unauthenticated remote code execution. Both were disclosed and added to the KEV catalog on October 31, 2023. [Praetorian researchers](https://www.praetorian.com/blog/refresh-compromising-f5-big-ip-with-request-smuggling-cve-2023-46747/) who discovered the chain noted it could be exploited by a network-adjacent attacker with no prior credentials; a public PoC was available within days of disclosure.

### CVE-2025-53521 — BIG-IP (KEV: 2026-03-27)

[CVE-2025-53521](https://nvd.nist.gov/vuln/detail/CVE-2025-53521) is a recent addition, cataloged in March 2026. Details on exploitation scope and mechanism are still accumulating.

## Transparency Assessment

F5 publishes security advisories through its K-article system (support.f5.com), with articles assigned a severity rating and a clear affected-version matrix. Advisory cadence has generally been timely: CVE-2022-1388, for example, was disclosed with patches available on the same day. There are no documented episodes of silent patching analogous to Fortinet's XORtigate firmware push. F5's disclosure practice is a relative strength in this peer group — the primary criticism is the speed at which attackers operationalize PoCs after F5's public advisories, which reflects the attractiveness of the target rather than a disclosure process failure.

## Risk Summary

6 KEV entries is moderate in absolute terms, but the severity profile is concentrated at the high end: three CVSS 9.8 entries (CVE-2020-5902, CVE-2022-1388, CVE-2023-46747), all enabling unauthenticated remote code execution or equivalent impact on internet-facing management interfaces. CVE-2022-1388 stands out for its exploitation velocity — two days from PoC to mass compromise across thousands of organizations. Defenders running BIG-IP with management or APM interfaces exposed to the internet should treat any critical F5 advisory as an emergency patch event, not a routine update cycle. The 2023 Config Utility chain reinforces that F5's risk is not limited to a single vulnerable component.
