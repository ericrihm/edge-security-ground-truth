# Data Dictionary — Edge Security Ground Truth

This document describes every field in the two canonical data files:

- `scripts/edge_kev_dataset.csv` — machine-readable flat export (one row per CVE)
- `scripts/kev_edge_enriched.json` — structured enrichment source (vendor → CVE → fields)

Dataset: 115 CISA KEV-listed CVEs across 13 edge-security vendors (2021–2026 KEV additions).

---

## 1. `scripts/edge_kev_dataset.csv`

Generated by `scripts/export_dataset.py`. One header row, 115 data rows.
All dates are ISO 8601 (`YYYY-MM-DD`). Empty string means the value is unavailable.

| Column | Type | Nullable | Source | Description |
|--------|------|----------|--------|-------------|
| `vendor` | string | No | `kev_edge_counts.json` | Vendor name as used in this dataset. One of 13 values: Array Networks, Check Point, Cisco, Citrix, F5, Fortinet, Ivanti, Juniper, Palo Alto Networks, SonicWall, Sophos, WatchGuard, Zyxel. |
| `cve` | string | No | `kev_edge_counts.json` | CVE identifier in standard form (`CVE-YYYY-NNNNN`). Unique within the dataset. |
| `kev_date_added` | date | No | CISA KEV catalog | Date CISA added this CVE to the Known Exploited Vulnerabilities catalog. All 115 entries have this value (kev_found=115). Format: `YYYY-MM-DD`. Range: 2021-11-03 – 2026-06-08. |
| `kev_due_date` | date | No | CISA KEV catalog | CISA-mandated federal agency remediation deadline. All 115 entries have this value. Format: `YYYY-MM-DD`. |
| `published` | date | No | NVD / CISA KEV | NVD public disclosure date (`publishedDate`), or the KEV `dateAdded` when NVD data was not fetched (see `_source` in enriched JSON). All 115 entries have this value. Format: `YYYY-MM-DD`. Range: 2014-03-19 – 2026-06-08. |
| `tte_days` | integer | No | Computed | Time-to-exploit in days: `kev_date_added − published`. Negative values indicate the CVE was added to KEV before its NVD publication date — a strong signal of zero-day exploitation. Range in this dataset: -25 – 3891. Present for all 115 rows (no blanks). |
| `cvss` | float | Yes | NVD (via `enrich_nvd.py`) | CVSS v3.x base score (0.0 – 10.0). Blank for 4 CVEs published too recently to have an NVD CVSS assignment at time of data generation. |
| `cvss_severity` | string | Yes | NVD (via `enrich_nvd.py`) | CVSS v3.x severity label corresponding to `cvss`. One of: `LOW` (< 4.0), `MEDIUM` (4.0–6.9), `HIGH` (7.0–8.9), `CRITICAL` (9.0–10.0). Blank when `cvss` is blank. |
| `cwe` | string | Yes | NVD (via `enrich_nvd.py`) | Primary CWE identifier (e.g., `CWE-89`). Blank for 2 CVEs where NVD did not supply a CWE classification at time of generation. |
| `cwe_name` | string | Yes | NVD (via `enrich_nvd.py`) | Human-readable name of the primary CWE (e.g., `Improper Neutralization of Special Elements used in an SQL Command`). Blank when `cwe` is blank. |
| `epss` | float | No | FIRST EPSS API (`enrich_epss.py`) | Exploit Prediction Scoring System probability score (0.0 – 1.0). Higher values indicate greater predicted likelihood of exploitation in the wild within 30 days. All 115 CVEs have an EPSS score. Range: 0.00865 – 0.99999. |
| `percentile` | float | No | FIRST EPSS API (`enrich_epss.py`) | EPSS percentile rank (0.0 – 1.0) indicating the fraction of all scored CVEs with a lower EPSS score. All 115 CVEs have this value. Range: 0.539 – 1.000. |
| `ransomware` | string | No | CISA KEV catalog | Whether this CVE has known use in ransomware campaigns. Values: `Known` (47 CVEs) or `Unknown` (68 CVEs). Sourced from CISA KEV `knownRansomwareCampaignUse` field; `Known` = "Known", all other values mapped to `Unknown`. |

### Notes

- Row ordering: vendor-alphabetical (Array Networks → Zyxel), then original build order within each vendor.
- The `cvss_vector` field from the enriched JSON is intentionally omitted from the CSV to keep it tabular; it is available in `kev_edge_enriched.json`.
- The `cwe_secondary` list field is also omitted from the CSV for the same reason.

## 2. `scripts/kev_edge_enriched.json`

Structured JSON with vendor-namespaced CVE entries plus dataset metadata. Generated by the enrichment pipeline (`build_kev_counts.py` → `enrich_epss.py` → `enrich_nvd.py` → `enrich_kev.py`).

### Top-level structure

```json
{
  "VendorName": {
    "CVE-YYYY-NNNNN": { ...fields... },
    ...
  },
  "_metadata": { ...dataset metadata... }
}
```

### Per-CVE fields

| Field | Type | Nullable | Source | Description |
|-------|------|----------|--------|-------------|
| `epss` | float | No | FIRST EPSS API | Exploit Prediction Scoring System probability (0.0 – 1.0). |
| `percentile` | float | No | FIRST EPSS API | EPSS percentile rank among all scored CVEs (0.0 – 1.0). |
| `cvss` | float \| null | Yes | NVD | CVSS v3.x base score. `null` for 4 recently published CVEs with no NVD score yet. |
| `cvss_severity` | string \| null | Yes | NVD | CVSS v3.x severity label (`LOW`/`MEDIUM`/`HIGH`/`CRITICAL`). `null` when `cvss` is `null`. |
| `cvss_vector` | string \| null | Yes | NVD | Full CVSS v3.x vector string (e.g., `CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N`). `null` when `cvss` is `null`. Not exported to CSV. |
| `cwe` | string \| null | Yes | NVD | Primary CWE identifier. `null` for 2 CVEs with no NVD CWE assignment. |
| `cwe_name` | string \| null | Yes | NVD | Human-readable primary CWE name. `null` when `cwe` is `null`. |
| `cwe_secondary` | array \| absent | Yes | NVD | List of secondary CWE identifiers (e.g., `["CWE-250"]`). Field absent when there are no secondary CWEs. Not exported to CSV. |
| `published` | string | No | NVD / CISA KEV | NVD public disclosure date (`YYYY-MM-DD`). When NVD data was not fetched, this is the CISA KEV `dateAdded` (see `_source`). |
| `kev_date_added` | string | No | CISA KEV catalog | Date CISA added this CVE to the KEV catalog (`YYYY-MM-DD`). |
| `kev_due_date` | string | No | CISA KEV catalog | CISA-mandated federal agency remediation deadline (`YYYY-MM-DD`). |
| `ransomware` | string | No | CISA KEV catalog | `"Known"` or `"Unknown"` ransomware campaign association. |
| `_source` | string \| null | Yes | Pipeline internal | Indicates the provenance of the `published` date. `null` when NVD was the source (normal case); `"kev"` when the KEV `dateAdded` was used as a fallback because NVD data was unavailable. |

### `_metadata` object

| Field | Type | Description |
|-------|------|-------------|
| `sources` | array of strings | URLs of upstream data sources used during enrichment. |
| `generated_at` | string (ISO 8601) | Timestamp when EPSS enrichment was last run. |
| `total_cves` | integer | Total CVE count in the dataset (115). |
| `nvd_fetched` | integer | Number of CVEs for which NVD data was fetched in the last enrichment run. |
| `nvd_errors` | integer | Number of NVD fetch errors. |
| `nvd_skipped` | integer | Number of CVEs skipped because NVD data was already present. |
| `epss_found` | integer | Number of CVEs for which EPSS scores were retrieved. |
| `epss_missing` | integer | Number of CVEs with no EPSS score (0 in current dataset). |
| `kev_enriched_at` | string (ISO 8601) | Timestamp when KEV enrichment was last run. |
| `kev_source` | string (URL) | CISA KEV catalog URL used for enrichment. |
| `kev_total_cves` | integer | Total CVEs processed during KEV enrichment (115). |
| `kev_found` | integer | CVEs matched in the CISA KEV catalog (115). |
| `kev_missing` | integer | CVEs not found in the CISA KEV catalog (0). |

## Reproducibility

All enrichment scripts are stdlib-only Python 3 (no pip dependencies):

```
python3 scripts/build_kev_counts.py       # build kev_edge_counts.json
python3 scripts/enrich_epss.py > scripts/kev_edge_enriched.json
python3 scripts/enrich_nvd.py > scripts/kev_edge_enriched.json
python3 scripts/enrich_kev.py > scripts/kev_edge_enriched.json
python3 scripts/export_dataset.py         # produce edge_kev_dataset.csv
```

The CSV export is fully deterministic given fixed input JSON files.
