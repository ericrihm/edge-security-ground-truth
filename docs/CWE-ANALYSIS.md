## Key Findings

1. **Auth is the dominant weakness class for edge devices.** 22% of exploited CVEs are authentication/authorization bypasses — the most consequential category because they grant unauthenticated remote access.

2. **Injection remains systemic.** SQL injection, OS command injection, and code injection collectively match auth flaws at 22%. Edge appliances built on Linux with shell-based management interfaces are structurally exposed to command injection.

3. **Memory safety affects every vendor.** 18% of exploited CVEs are buffer overflows, heap corruptions, or out-of-bounds writes. Vendors shipping C/C++ network stacks without modern memory-safe alternatives inherit this class of risk.

4. **Vendor-specific patterns are actionable.** Fortinet's concentration in auth bypasses (7 CVEs) and Ivanti's in injection (6 CVEs) suggest specific SDL gaps that defenders can anticipate in vulnerability management prioritization.

## Methodology

CWE assignments come from NVD (primary) and KEV catalog (fallback). Categories are mapped using a curated taxonomy of 30+ CWEs grouped into 10 categories. The "recurring weakness" threshold of 3+ CVEs in a single category per vendor was chosen to balance signal (enough CVEs to indicate a pattern) against noise (any vendor with enough CVEs will hit some categories by chance).
